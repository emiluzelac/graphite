# Glass UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make frosted glass ("Frost": alpha surface + 16px backdrop-blur + hairline border) Graphite's default look, and restore Headless UI's render-prop `className` support in every wrapper.

**Architecture:** The material is defined once as `glass`/`glass-flat` Tailwind v4 `@utility` blocks driven by new `--glass-*` tokens that flip with `.dark` (components keep zero `dark:` variants). A `Backdrop` component provides the ambient gradient stage. Wrappers switch from `Omit<Props, 'className'> & { className?: string }` to Headless UI's native `string | (bag) => string` className, resolved through a shared `composeClass` helper.

**Tech Stack:** React 19, @headlessui/react 2.2, Tailwind CSS 4 (`@utility`, alpha OKLCH), CVA, vitest + testing-library (jsdom), shadcn registry CLI.

**Spec:** `docs/superpowers/specs/2026-07-06-glass-ui-design.md` — read it first; it is the source of truth for every value below.

## Global Constraints

- Components must contain **zero `dark:` variants** — tokens do all mode switching.
- Existing shadcn slots (`--popover`, `--card`, …) keep their opaque values; glass components stop referencing them. Text tokens (`foreground`, `muted-foreground`) unchanged.
- Glass token values (verbatim from spec): light `--glass-surface: oklch(1 0 0 / 0.55)`, `--glass-border: oklch(0.21 0.016 295 / 0.1)`, `--glass-shadow: oklch(0.21 0.016 295 / 0.08)`, `--glass-blur: 16px`; dark `--glass-surface: oklch(1 0 0 / 0.08)`, `--glass-border: oklch(1 0 0 / 0.12)`, `--glass-shadow: oklch(0 0 0 / 0.25)`.
- State fills on glass surfaces use foreground-alpha: focus/active `bg-foreground/10`, hover `bg-foreground/5`. Never opaque `accent`/`secondary` on glass.
- TDD: failing test before implementation for every new contract. Run: `npx vitest run` (full suite), `npm run build` (tsc + vite), `npm run lint`, `npm run format:check`. All four must pass before every commit.
- After changing component sources or `registry.json`: `npm run registry:build && npx prettier --write public/r/ --log-level warn`.
- Commit per task (the spec's "three commits" are realized as three ordered stages, one commit per task within them). End commit messages with:
  `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`
- Interpretation note (approved in design review): the selected Radio card uses `data-checked:bg-foreground/10` (state fill on a glass surface), not opaque `bg-secondary`. The spec's "checked … fills stay solid" line applies to primary-colored fills (Checkbox check, Switch track).

- [ ] **Step 3: Implement in `src/lib/cn.ts`**

Append:

```ts
/**
 * Resolve a Headless UI className prop (string or render-prop function)
 * against wrapper base classes. Consumer classes win conflicts via
 * tailwind-merge. Always returns a function so Headless UI supplies the
 * state bag.
 */
export function composeClass<Bag>(
  base: string,
  className?: string | ((bag: Bag) => string),
): (bag: Bag) => string {
  return (bag) => cn(base, typeof className === 'function' ? className(bag) : className)
}
```

- [ ] **Step 1: Add a "Glass material" section to `docs/design-tokens.md`**

Insert after the Architecture section:

```markdown
## Glass material

Graphite's default surface treatment is frosted glass ("Frost"): alpha surface +
16px backdrop-blur + hairline border. It is defined once as two utilities in
`src/index.css`, driven by four tokens that flip with `.dark`:

| Token             | Light                          | Dark                  |
| ----------------- | ------------------------------ | --------------------- |
| `--glass-surface` | `oklch(1 0 0 / 0.55)`          | `oklch(1 0 0 / 0.08)` |
| `--glass-border`  | `oklch(0.21 0.016 295 / 0.1)`  | `oklch(1 0 0 / 0.12)` |
| `--glass-shadow`  | `oklch(0.21 0.016 295 / 0.08)` | `oklch(0 0 0 / 0.25)` |
| `--glass-blur`    | `16px`                         | (same)                |

- `glass` — floating surfaces (menus, popovers, dialogs, option lists): surface,
  hairline border, `backdrop-filter: blur(...) saturate(120%)`, soft shadow.
- `glass-flat` — inline controls (inputs, triggers, cards): surface + border only.

**State fills on glass use foreground-alpha**, never opaque `accent`/`secondary`
(opaque fills punch holes in translucent panels): focus/active `bg-foreground/10`,
hover `bg-foreground/5`. `--foreground` flips per mode, so this is white-alpha in
dark and ink-alpha in light.

The shadcn slots (`--popover`, `--card`, …) keep opaque values for contract
compatibility; glass components do not reference them. The optional `backdrop`
registry item provides the ambient gradient stage (`--backdrop-1/2/3`) the demos
sit on. Without `backdrop-filter` support, surfaces degrade to plain alpha film.
```
